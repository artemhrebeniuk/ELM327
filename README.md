# 🚘 ELM327 Smart Dashboard & Adaptive Video Player

> **Напутствие потомкам:** Этот репозиторий — плод борьбы с капризным китайским железом ELM327 и устаревшими графическими библиотеками macOS. Если вы читаете это, значит, перед вами стоит задача подружить Python с бортовым компьютером автомобиля (ЭБУ) по протоколу OBD-II или низкоуровневым UDS-запросам, а также визуализировать данные телеметрии. Берегите этот код.

Данный репозиторий представляет собой **экосистему из двух тесно связанных приложений**, которые могут работать как раздельно, так и быть скомпилированными в **одну единую программу** для удобства:
1. **OBD-II Data Scanner** (`obd_gui_qt.py`) — Считывает физические данные с автомобиля в реальном времени.
2. **Adaptive Video Player** (`video_player.py`) — Синхронизирует воспроизведение видео с вашей скоростью езды, используя данные сканера.

Обе программы общаются между собой по локальной сети через протокол **UDP**, что позволяет им работать асинхронно, не блокируя друг друга.

---

## 🛠 Проект 1: OBD-II Data Scanner (`obd_gui_qt.py`)

Служит мостом между железом автомобиля и вашим компьютером. Читает данные через адаптер ELM327 (Wi-Fi или Bluetooth), подключенный в разъем OBD-II.

### ✨ Ключевые возможности сканера
* **Двойной профиль автомобиля:** Поддержка стандартных ДВС машин через OBD-II (PID Mode 01) и электромобилей (например, Audi E-tron / VAG EV) через низкоуровневые UDS-запросы (Service 0x22).
* **Премиальный UI на PyQt5:** Неоновое свечение, плавная интерполяция стрелок с 60 FPS через экспоненциальное сглаживание (QTimer), поддержка Retina.
* **Адаптивный интерфейс:** Панели приборов масштабируются автоматически, предотвращая обрезку текста и элементов при изменении размеров окна.
* **Запись телеметрии и чтение ошибок (DTC):** Фоновое логирование показателей (скорость, заряд, обороты, мощность) в CSV-файл и опрос памяти ошибок ЭБУ с записью в файл отчета.
* **Demo Simulator:** Встроенный программный симулятор `elm_emulator.py` для отладки UI дома.

## 🎥 Проект 2: Adaptive Video Player (`video_player.py`)

Уникальный видеоплеер на базе PyQt5 и OpenCV, который считывает телеметрию от сканера и **напрямую привязывает скорость воспроизведения видео к вашей реальной скорости**. Идеально для демонстраций, где видео должно идти синхронно с движением машины.

### 🧠 Логика работы плеера и новые функции
1. **100% Идеальное качество в ОЗУ (JPEG Cache):** Плеер загружает видео-файл через OpenCV и сохраняет каждый его кадр в оперативную память, сжимая в формат JPEG со 100% качеством. Это позволяет программе молниеносно прыгать по кадрам без потери визуального качества, при этом защищая от краша нехватки памяти (OOM) на больших видео.
2. **Премиальный анимированный экран загрузки:** Во время кэширования видео отображается стильный глоу-эффект, процент выполнения, размер кэша и современный векторный спиннер.
3. **Связь со сканером (UDP-линк):** Сканер непрерывно отправляет пакеты (например, `SPEED:60.5`) на порт `28765`. Встроенный в плеер `UdpListener` ловит их и плавно интерполирует кадры.
4. **Проекция во внешнее окно (Borderless Window):** Специальная кнопка "PROJECTION" открывает видео в отдельном окне без рамок (frameless window). Это окно можно перетаскивать за любую точку, растягивать и разворачивать на весь экран нажатием клавиши `F`. Идеально для вывода видео на второй монитор (например, проектор на лобовое стекло) без элементов управления поверх.

### 🎮 Режимы воспроизведения
* **Reversible (Dynamic):** Интерактивный режим "туда-обратно". Разгоняетесь — видео играет вперед. Нажали на тормоз — **видео проигрывается задом наперед**, отматывая время до тех пор, пока машина не остановится.
* **Forward Only (Classic):** Классический режим. Торможение машины просто замедляет видео. Остановка машины ставит видео на паузу.
* **Autoplay (Loop):** Третий автономный режим. Видео циклично проигрывается само по себе без привязки к скорости авто. Появляются кнопки PLAY/PAUSE, а ползунок управляет множителем скорости (0.1x — 2.0x).

## 🚀 Инструкция по запуску (Из исходников)

Если вы запускаете код из Python-скриптов:

1. **Подключите ELM327** в ваш автомобиль (или запустите `python3 elm_emulator.py`).
2. Откройте Терминал и **запустите плеер**:
   ```bash
   python3 video_player.py
   ```
3. Прямо в интерфейсе плеера нажмите **Launch OBD Scanner**. Программа автоматически запустит `obd_gui_qt.py` вторым окном. Связь по UDP установится сама.

## 📦 Сборка в одну программу (.exe для Windows)

Мы спроектировали архитектуру так, чтобы обе программы собирались в **ОДИН** исполняемый файл. Плеер и сканер живут внутри одного `.exe`.
* При запуске файла по умолчанию открывается **Adaptive Video Player**.
* При нажатии на кнопку **"Launch OBD Scanner"** плеер вызывает сам себя (`sys.executable`) с аргументом `--run-obd-scanner` и рабочей директорией (`cwd`).
* Процесс запускает второе независимое окно — **OBD-II Scanner**. Они мгновенно связываются по UDP на порту `28765`.

### Команда для компиляции:
Для корректного отображения иконок внутри `.exe` необходимо принудительно запаковать файл логотипа `logo.png`.

1. Установите **PyInstaller**:
   ```cmd
   pip install --upgrade pyinstaller
   ```
2. Выполните команду компиляции в папке проекта:
   * **Для Windows** (используется разделитель `;`):
     ```cmd
     pyinstaller --onefile --clean --name "ELM327_System" --add-data "logo.png;." video_player.py
     ```
   * **Для macOS** (используется разделитель `:`):
     ```bash
     pyinstaller --onefile --clean --name "ELM327_System" --add-data "logo.png:." video_player.py
     ```
3. Готовый файл `ELM327_System.exe` (или `ELM327_System` на Mac) появится в папке `dist/`.

> [!IMPORTANT]
> **Почему не используется флаг `--noconsole`?**
> Мы намеренно оставили консоль активной. Низкоуровневые библиотеки (включая `obd` для работы с ELM327) могут перехватывать стандартные потоки вывода (`stdout`/`stderr`) или падать при отсутствии консольного контекста в Windows. Кроме того, в консоли выводятся важные отладочные логи подключения к ЭБУ.

## 🛠 Техническая информация для разработчиков и AI-моделей

### 1. Архитектура нулевой задержки (Zero-Latency)
Чтобы минимизировать задержку считывания данных:
- Мы убрали передачу сигналов через очередь событий главного UI-потока.
- `UdpListener` слушает порт `28765` и при получении пакета **напрямую** вызывает метод `update_speed()` в `PlaybackThread`.
- Доступ защищен `threading.Lock()` мьютексом, обеспечивая полную потокобезопасность.

### 2. Алгоритм сглаживания и компенсации задержек адаптера
Реальные OBD-адаптеры могут передавать данные редко (с частотой 1-2 Гц). Чтобы видео не шло рывками:
- **Интерполяция интервала**: Программа замеряет скользящее среднее время между пакетами (`average_packet_interval`) и плавно дотягивает скорость видео до целевой.
- **Экспоненциальное сглаживание (EMA)**: Фильтрует шумы датчиков. Коэффициент сглаживания регулируется ползунком **SMOOTHING** (от `0.01` до `1.00`). При значении `1.00` сглаживание отключается для мгновенного отклика.
- **Защита от сбоев связи**: Тайм-аут сокета выставлен на `2.0` секунды. Это защищает от ложных сбросов скорости на медленных адаптерах.

### 3. Добавление новых параметров в OBD-сканер
Если в будущем вам нужно добавить новый датчик в `obd_gui_qt.py`:
1. **Для EV (UDS-запросы):** Добавьте HEX-идентификатор (DID) в список запросов в потоке `poll_obd_data` и пропишите формулу разбора в `_decode_response()`.
2. **Для ДВС (OBD-II):** Убедитесь, что PID поддерживается стандартом SAE J1979, запросите его через `obd.commands.YOUR_COMMAND`.
3. Передайте значение через сигнал `self.signals.update_data.emit(...)` и выведите его на GUI-прибор.

*Пусть код работает без сбоев, а коннект всегда будет стабильным!*
