## 🌟 Описание проекта
Приложение предназначено для реального времени считывания параметров работы автомобиля через адаптер ELM327 (Wi-Fi / Bluetooth), подключенный в разъем OBD-II. 

**✨ Ключевые возможности программы (Последнее обновление):**
* **Двойной профиль автомобиля (Vehicle Profiles):**
  * **Audi E-tron / VAG EV**: Профиль для электромобилей. Запросы идут через низкоуровневый протокол **UDS Service 0x22 (Read Data By Identifier)**.
    * Выводит: **Мощность батареи в кВт** (с динамической индикацией: голубой — рекуперация, желтый — покой, оранжево-красный — активный разгон), **Заряд батареи SOC (%)**, **Температуру батареи (°C)** и скорость.
    * Включает фазу сканирования: перебирает популярные CAN-заголовки (7E0, 7E4, 7DF, 7E5, 714, 720) для нахождения ЭБУ батареи (BMS).
  * **Mercedes Sprinter / Standard ICE**: Профиль для автомобилей с ДВС. Запросы идут через **Standard OBD-II PIDs**.
    * Выводит: **Обороты двигателя (RPM)**, **Напряжение бортовой сети (V)**, **Температуру антифриза (°C)** и скорость.
* **Разовое чтение пробега (Odometer):** При успешном подключении программа отправляет UDS-запросы к приборной панели (адрес `714`) или блоку двигателя (`7E0`) с кодами `222203`, `222205` и `220902`. Она декодирует и выводит реальный общий пробег автомобиля в километрах прямо в боковую панель.
* **Премиальный UI на PyQt5:**
  * **Аппаратная отрисовка (`QPainter`):** Круговые и линейные шкалы с неоновым свечением.
  * **Сглаженный Glow-эффект:** Эффект свечения прорисовывается за 15 микрошагов (по 2px) с квадратичным угасанием альфа-канала, имитируя мягкое аналоговое размытие без полос и стыков.
  * **Компактная верстка настроек:** Все элементы сгруппированы в блоки с отступом `4px` (лейблы прижаты к полям ввода), а кнопка **Scan** расположена на одной строке с выбором COM-порта. Это предотвращает выход интерфейса за границы экрана на ноутбуках.
  * **Современная темная тема:** Однотонный плоский фон сайдбара (`#12141d`) гармонирует с карточками. Выпадающие списки избавлены от белых macOS-рамок с помощью форсирования Qt-отрисовки (`combobox-popup: 0`), а выбор элемента подсвечивается мягким серым фоном и бирюзовым текстом.
* **Запись телеметрии (CSV Logger):** Сохраняет показатели поездки в фоновом режиме в формат `.csv`.
* **Считывание ошибок (DTC Scanner):** Записывает коды неисправностей двигателя в текстовый лог при включенной галочке.
* **Demo Simulator:** Встроенный генератор данных для тестирования интерфейса без физического подключения к адаптеру.

## 🚀 Тестирование с помощью Эмулятора (Без машины)

Вы можете протестировать всю цепочку соединения (как в режиме ДВС, так и в UDS EV режиме) прямо на компьютере:

### Шаг 1. Создание виртуального моста (Null-Modem)
*   **На Windows**: Используйте утилиту **com0com** или **VSPE** для создания связанной пары портов (например, `COM10` <-> `COM11`).
*   **На macOS/Linux**: Используйте команду `socat` для связывания двух tty-портов.
    ```bash
    socat -d -d pty,raw,echo=0 pty,raw,echo=0
    ```
    Команда выведет пути к двум созданным tty (например, `/dev/ttys001` и `/dev/ttys002`).

### Шаг 2. Запуск эмулятора
Эмулятор займет первый порт (например, `COM10` или `/dev/ttys001`):
```bash
python elm_emulator.py COM10
```

### Шаг 3. Запуск приборной панели
1. Запустите панель:
   ```bash
   python obd_gui_qt.py
   ```
2. **Снимите галочку** с *"Demo Simulator"*.
3. Выберите порт **`COM11`** (или `/dev/ttys002`) и нажмите **Connect**.
4. Протестируйте переключение между профилями **VAG EV** и **ICE** — эмулятор динамически адаптирует свои ответы и будет имитировать разгоны, торможения, изменение вольтажа и рекуперацию заряда батареи!

## 🔌 Работа с реальным ELM327: Нюансы железа и связи

### 1. Подключение по Wi-Fi (TCP socket)
* **Адрес по умолчанию:** `socket://192.168.0.10:35000` (стандарт для большинства OBD-адаптеров).
* **Оптимизация битрейта:** Для Wi-Fi скорость порта зафиксирована в коде (`baudrate = 38400`), чтобы исключить зависания при автоматическом переборе битрейтов в библиотеке `python-obd`.

### 2. Подключение по Bluetooth (Serial / RFCOMM)
* **Совместимость:** Поддерживаются только классические OBD2 Bluetooth V1.5 адаптеры (например, **Kingbolen OBD SCAN**). Устройства *OBDeleven* использовать не получится из-за закрытого BLE-протокола.
* **Сопряжение в macOS:** Выполните сопряжение в настройках macOS (обычно PIN `1234`). Система создаст порт `/dev/cu.OBDII`. В выпадающем списке портов используйте именно `/dev/cu.*` (он не требует сигнала DCD и не зависает при открытии в macOS, в отличие от `/dev/tty.*`).
* **Предварительная BT диагностика (`_test_bt_raw`):** В коде реализована функция низкоуровневой проверки порта перед инициализацией `python-obd`. Программа открывает порт напрямую, посылает тестовую команду сброса `ATZ` и ждет ответа. Если ответа нет, приложение выдаст детальные подсказки по питанию адаптера и настройкам ОС, предотвращая бесконечное зависание.
* **Таймаут чтения:** Увеличен до `8.0` секунд для компенсации пингов Bluetooth.
* **Пауза сброса канала (1.5 сек):** При сбое подключения программа выжидает 1.5 сек перед переподключением, давая ОС освободить порт.

### 3. Оптимизация цикла опроса
Чтобы обеспечить максимальную скорость обновления (близкую к реальному времени), из постоянного опроса EV-профиля был удален неиспользуемый параметр **Range (Запас хода)**. Напряжение и ток опрашиваются непрерывно для расчета мощности в кВт. Задержка в цикле опроса составляет оптимальные `150 мс`, чтобы не перегружать буфер слабых ELM327 клонов.

## 🔮 Напутствие будущим разработчикам

Если потребуется добавить новые параметры:
1. Добавьте нужные DIDs в списки перебора (например, `VOLTAGE_DIDS_TO_TRY`).
2. При необходимости добавьте декодирование сырых байт ответа в методе `_decode_response`.
3. Не перегружайте цикл опроса слишком большим количеством параметров одновременно, иначе частота обновления приборов на экране снизится.

*Да пребудет с вами стабильный коннект, чистый лог и надежный Bluetooth-сигнал!*
