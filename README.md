## 🌟 Описание проекта
Приложение предназначено для реального времени считывания параметров работы автомобиля через адаптер ELM327 (Wi-Fi / Bluetooth), подключенный в разъем OBD-II. 

**✨ Ключевые возможности программы (Последнее обновление):**
* **Двойной профиль автомобиля (Vehicle Profiles):**
  * **Audi E-tron / VAG EV**: Профиль для электромобилей. Запросы идут через низкоуровневый протокол **UDS Service 0x22 (Read Data By Identifier)**.
    * Выводит: **Мощность батареи в кВт** (с цветовой кодировкой: голубой — рекуперация, зеленый — покой, оранжево-красный — разгон), **Заряд батареи SOC (%)**, **Температуру батареи (°C)** и скорость.
    * Включает фазу сканирования: перебирает популярные CAN-заголовки (7E0, 7E4, 7DF, 7E5, 714, 720) для нахождения ЭБУ батареи (BMS).
  * **Mercedes Sprinter / Standard ICE**: Профиль для автомобилей с ДВС. Запросы идут через **Standard OBD-II PIDs**.
    * Выводит: **Обороты двигателя (RPM)**, **Напряжение бортовой сети (V)** (вместо уровня топлива в четвертом приборе), **Температуру антифриза (°C)** и скорость.
* **Премиальный UI:** Разработан уникальный интерфейс на базе аппаратной отрисовки (`QPainter`). Сглаженные круговые и линейные шкалы, неоновое свечение (Glow), засечки, аппаратные эффекты парения (3D Drop Shadows) и градиентный сайдбар — всё как в топовых цифровых панелях спорткаров.
* **Запись телеметрии (CSV Logger):** Сохраняет показатели поездки в фоновом режиме в формат `.csv` для последующего анализа и построения графиков расхода батареи.
* **Считывание ошибок (DTC Scanner):** Отдельный функционал сканирования блока ЭБУ на наличие ошибок двигателя. Результаты сканирования записываются в текстовый лог.
* **Demo Simulator:** Встроенный генератор данных для демонстрации интерфейса без машины. Имитирует разгон, рекуперацию, переключение передач и случайные ошибки (Check Engine).

Проект спроектирован так, чтобы работать без зависаний интерфейса (опрос вынесен в фоновый поток) и поддерживать как Windows, так и macOS.

## 🚀 Тестирование с помощью Эмулятора (Без машины)

Вы можете протестировать всю цепочку соединения (включая опрос COM-портов и декодирование OBD-протокола) прямо на компьютере:

### Шаг 1. Создание виртуального моста (Null-Modem)
Для передачи данных нам нужна виртуальная связанная пара портов (например, `COM10` и `COM11`).
*   **На Windows**: Скачайте и установите бесплатную утилиту **com0com** или **VSPE** (создайте пару типа Connector/Bridge).
*   **На macOS/Linux**: Можно использовать команду `socat` для связывания двух tty-портов.

### Шаг 2. Запуск эмулятора
Эмулятор займет порт `COM10` (или укажите ваш порт аргументом при запуске):
```bash
python elm_emulator.py COM10
```
*Эмулятор начнет ждать подключения со стороны нашего GUI.*

### Шаг 3. Запуск приборной панели
1. Запустите графическую панель приборов:
   ```bash
   python obd_gui_qt.py
   ```
2. **Снимите галочку** с *"Demo Simulator Mode"*.
3. Выберите из списка порт **`COM11`** (второй конец виртуального моста) и нажмите **Connect**.
4. На приборной панели стрелки и цифры начнут плавно колебаться, считывая волнообразные сигналы с эмулятора, а в консоли эмулятора вы увидите подробный лог OBD-запросов.

## 🔌 Работа с реальным ELM327: Нюансы железа и беспроводной связи

При подключении к реальному автомобилю (особенно по Wi-Fi или Bluetooth) мы исправили и учли важные технические аспекты:

### 1. Подключение по Wi-Fi (TCP socket)
* **IP по умолчанию:** `socket://192.168.0.10:35000` (стандарт для Wi-Fi OBD адаптеров).
* **Фикс автобитрейта:** Для Wi-Fi соединений скорость порта принудительно зафиксирована в коде (`baudrate = 38400`), чтобы исключить запуск функции `auto_baudrate()` в `python-obd`. Это решает проблему отправки мусорных байт в TCP-сокет и исключает зависание сканера при переборе протоколов с ошибкой `Failed to set baudrate`.

### 2. Подключение по Bluetooth (Serial / RFCOMM)
* **Совместимость устройств:** Используйте только классические OBD2 Bluetooth V1.5 приборы (например, **Kingbolen OBD SCAN**). Устройства типа *OBDeleven* использовать не получится, так как они работают по закрытому зашифрованному BLE-протоколу.
* **Сопряжение в macOS:** Сначала сопрягите адаптер в системных настройках Bluetooth (PIN обычно `1234` или `0000`). macOS создаст виртуальный порт `/dev/tty.OBDII`.
* **Автовыбор скорости `38400`:** При выборе Bluetooth-режима программа автоматически переключает скорость на `38400`. Это предотвращает сбои виртуального последовательного порта при попытках автосканирования скоростей.
* **Таймаут чтения (1.5 сек):** Bluetooth-соединение имеет физическую задержку (latency). Мы увеличили таймаут чтения в `python-obd` до `1.5` секунд (по умолчанию в библиотеке стоит жесткие `0.1` сек). Это полностью убрало ошибки `Failed to read port` и `ATE0 did not return 'OK'`.
* **Пауза сброса канала (1.5 сек):** При сбое подключения программа выжидает 1.5 секунды, давая Bluetooth-стеку macOS полностью разорвать и освободить канал связи.
* **Обнаружение портов:** На macOS реализован прямой поиск портов в `/dev/tty.*`, содержащих `obd`/`elm`/`scan`/`kingbolen`. Виртуальный порт `/dev/tty.OBDII` гарантированно отобразится в выпадающем списке, даже если адаптер находится в режиме сна.

### 3. Фикс TypeError при работе со стандартными PIDs
Мы модифицировали код встроенной библиотеки `python-obd` ([OBDCommand.py](file:///Users/mr_shpepe/Library/Python/3.9/lib/python/site-packages/obd/OBDCommand.py) и [obd.py](file:///Users/mr_shpepe/Library/Python/3.9/lib/python/site-packages/obd/obd.py)). Это устранило ошибку `TypeError: can't concat NoneType to bytes`, возникавшую при опросе стандартных параметров (у которых нет CAN-заголовка, то есть `header = None`). Теперь заголовки `None` безопасно маппятся на функциональный заголовок `b"7DF"`.

## ⚠️ macOS: Проклятие системного Tcl/Tk
Если вы запустите версию на `customtkinter` (`obd_gui.py`) через системный `/usr/bin/python3`, вы увидите пустое серое окно и предупреждение:
`DEPRECATION WARNING: The system version of Tk is deprecated...`

**Почему это происходит:** 
Apple поставляет в комплекте с macOS древнюю версию библиотеки Tcl/Tk (8.5.9), которая некорректно работает с современным рендерингом окон.
**Решение:** 
Используйте **`obd_gui_qt.py`** (версию на Qt), либо установите современный Python c официального сайта [python.org](https://www.python.org/) (там Tcl/Tk версии 8.6+ уже встроен в установщик и работает отлично).

## 🔮 Напутствие будущим разработчикам

Если вам потребуется расширить список опрашиваемых параметров:
1. Найдите код нужной команды в таблице поддерживаемых PIDs библиотеки `python-OBD` (например, `obd.commands.COOLANT_TEMP` или `obd.commands.THROTTLE_POS`).
2. Добавьте опрос этой команды в фоновом цикле по аналогии с существующими датчиками.
3. Не опрашивайте слишком много датчиков в одном потоке последовательно — это снизит частоту обновления экрана (FPS). Для частых обновлений используйте асинхронные команды (`obd.Async`).

*Да пребудет с вами стабильный коннект, чистый лог и надежный Bluetooth-сигнал!*
