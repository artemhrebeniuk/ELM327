# 🚘 OBD-II ELM327 Smart Dashboard

> **Напутствие потомкам:** Этот репозиторий — плод борьбы с капризным китайским железом ELM327 и устаревшими графическими библиотеками macOS. Если вы читаете это, значит, перед вами стоит задача подружить Python с бортовым компьютером автомобиля (ЭБУ) по протоколу OBD-II. Ниже изложены все знания, архитектурные решения и подводные камни, собранные воедино. Берегите этот код.

---

## 🌟 Описание проекта
Приложение предназначено для реального времени считывания параметров работы двигателя (на примере **скорости автомобиля** и **оборотов двигателя RPM**) через адаптер ELM327 (Bluetooth/Serial), подключенный в разъем OBD-II. 

Проект спроектирован так, чтобы работать без зависаний интерфейса (опрос вынесен в фоновый поток) и поддерживать как Windows, так и macOS (включая обход системных багов рендеринга).

## 🚀 Запуск и использование

### 1. Демо-режим (Для тестирования без машины)
Оба графических скрипта содержат переключатель **Demo Simulator Mode**. 
1. Просто запустите скрипт:
   ```bash
   python obd_gui_qt.py
   ```
2. Оставьте галочку *Demo Simulator Mode* включенной и нажмите **Connect**.
3. Приложение начнет симулировать реальную поездку (разгон по передачам, падение оборотов при переключении, торможение), чтобы вы могли проверить анимацию и цвета приборов.

### 2. Подключение к реальному автомобилю
1. Вставьте адаптер ELM327 в OBD-II разъем автомобиля.
2. Включите зажигание (или заведите двигатель).
3. Создайте Bluetooth-пару между ПК и адаптером (код сопряжения обычно `1234` или `0000`).
4. Запустите программу, выберите ваш COM/tty порт из выпадающего списка (или используйте **Auto-Detect**).
5. Отключите *Demo Simulator Mode* и нажмите **Connect**.

## ⚠️ macOS: Проклятие системного Tcl/Tk
Если вы запустите версию на `customtkinter` (`obd_gui.py`) через системный `/usr/bin/python3`, вы увидите пустое серое окно и предупреждение:
`DEPRECATION WARNING: The system version of Tk is deprecated...`

**Почему это происходит:** 
Apple поставляет в комплекте с macOS древнюю версию библиотеки Tcl/Tk (8.5.9), которая некорректно работает с современным рендерингом окон.
**Решение:** 
Используйте **`obd_gui_qt.py`** (версию на Qt), либо установите современный Python c официального сайта [python.org](https://www.python.org/) (там Tcl/Tk версии 8.6+ уже встроен в установщик и работает отлично).

## 🔌 Работа с реальным ELM327: Нюансы железа

Когда вы начнете масштабировать проект, помните о следующих правилах работы с шиной OBD-II:

1.  **Проблема дешевых клонов ELM327 (v2.1)**: 
    Большинство дешевых китайских адаптеров поддерживают не все AT-команды. При подключении в коде используется флаг `fast=False` (`obd.OBD(fast=False)`). Это отключает оптимизацию групповых запросов OBD, предотвращая зависание дешевых чипов.
2.  **Задержка опроса (Polling Delay)**:
    Шина CAN в автомобиле не любит частых запросов. В бесконечном цикле чтения обязательно должна быть пауза не менее `0.1` секунды (`time.sleep(0.1)`). Иначе адаптер переполнит буфер и вернет пустой ответ или разорвет связь.
3.  **Отладка соединения (Debug Mode)**:
    Если связь не устанавливается, раскомментируйте следующую строку перед созданием подключения:
    ```python
    obd.logger.setLevel(obd.logging.DEBUG)
    ```
    В консоль начнет выводиться сырой лог общения с адаптером (запросы `ATZ`, `ATE0`, `ATSP` и ответы ЭБУ). Это поможет понять, на каком этапе происходит сбой.

## 🔮 Напутствие будущим разработчикам

Если вам потребуется расширить список опрашиваемых параметров (например, температура охлаждающей жидкости, давление наддува или ошибки Check Engine):
1. Найдите код нужной команды в таблице поддерживаемых PIDs библиотеки `python-OBD` (например, `obd.commands.COOLANT_TEMP`).
2. Добавьте опрос этой команды в фоновом цикле по аналогии со `SPEED` и `RPM`.
3. Не опрашивайте слишком много датчиков в одном потоке последовательно — это снизит частоту обновления экрана (FPS). Для частых обновлений используйте асинхронные команды (`obd.Async`).

*Да пребудет с вами стабильный коннект и чистый лог без ошибок тайм-аута!*
